# DESIGN

## Модель данных

- `approval_requests` — заявка, workspace, источник, список ревьюеров, статус и решение.
- `approval_audit_logs` — история успешных изменений статуса и создания заявки.
- `outbox_events` — события для будущей интеграции с брокером.

`reviewerUserIds` хранятся JSON-полем. Для тестового сервиса это проще, чем отдельная
таблица, а внешний идентификатор ревьюера все равно не валидируется внутри сервиса.

## Границы сервиса

Сервис отвечает только за lifecycle заявки на согласование. Он не проверяет существование
publication/scenario/edit/external и не ходит в соседние сервисы.

## Workspace isolation

Все чтения и изменения ищут заявку по паре `workspace_id + request_id`. Если заявка есть
в другом workspace, клиент получает `404`, без раскрытия факта существования объекта.

## Идемпотентность

Создание поддерживает заголовок `Idempotency-Key`. Ключ уникален в рамках workspace:
повтор `workspace_id + Idempotency-Key` возвращает уже созданную заявку и не пишет новые
audit/outbox записи. Без ключа создается новая заявка.

## Статусы

Стартовый статус — `pending`. Финальные статусы — `approved`, `rejected`, `cancelled`.
После финального статуса дальнейшие решения возвращают `409 Conflict`.

## Audit log

Audit trail фиксирует кто, когда и какое действие выполнил, а также старый и новый статус.
Это нужно для разборов решений и будущих админских экранов.

## Outbox events

`outbox_events` реализует outbox pattern: событие записывается в той же транзакции, что и
изменение заявки. Позже отдельный publisher сможет читать неопубликованные события и
отправлять их в Kafka/RabbitMQ, выставляя `published_at`.

## Защита секретов

Перед записью публичных текстовых полей и payload события используется простая redaction
функция. Она маскирует email, URL, bearer-токены и значения подозрительных ключей вроде
`secret`, `token`, `storage_key`, `signed_url`, `provider_url`.

## Компромиссы

- Auth реализован как локальная заглушка на HTTP-заголовках.
- Outbox только пишет события, но не публикует их.
- `reviewerUserIds` лежит JSON-полем для простоты.
- SQLite поддерживается для локального запуска, PostgreSQL рекомендуется через Docker Compose.
